# Assets

## Structure

```
assets/
├── screenshots/     # Product screenshots for README and Web Store
├── demo/            # Demo GIF and video clips
└── store/           # Chrome Web Store specific assets
```

## What to Add

### screenshots/

Capture real screenshots of the extension in action:

- `popup.png` — Extension popup window
- `element-highlight.png` — Blue overlay + tooltip on a webpage element
- `yaml-output.png` — Generated YAML output example
- `markdown-output.png` — Generated Markdown output example
- `multi-select.png` — Multiple elements with colored overlays
- `cmd-c-spec.png` — Full component spec generated by Cmd+C mode

### demo/

Create short screen recordings:

- `demo.gif` — 15–30 second loop showing the full workflow
  - Open page → activate extension → hover → click → copy → paste into AI chat

### store/

Chrome Web Store specific assets:

- `screenshot-1.png` through `screenshot-5.png` — 1280×800 or 640×400 screenshots
- `promo-marquee.png` — 1400×560 promotional image (optional)
- `icon-128.png` — Copy of the 128×128 extension icon

## Notes

- Do not include fake UI or mock data in screenshots
- Use real elements from well-known websites
- Blur or avoid sensitive content if necessary
- Keep screenshots clean — no annotations unless showing specific features
