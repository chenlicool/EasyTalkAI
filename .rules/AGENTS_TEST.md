# AGENTS_TEST.md (测试法典)

版本号: v1.0
生效时间: 2026-05-14
适用对象: 所有代码提交、功能验证、Bug 修复

## 1. 适用范围 (Scope)
所有项目功能的验证流程、测试用例编写。

## 2. 强制规则 (MUST)
- **AI 审计**：AI 生成或修改的代码默认视为"未可信"。
- **手动验证**：Chrome 扩展无法自动化测试，每次改动必须在浏览器中手动验证核心路径：
  1. Click 单元素捕获
  2. Shift+Click 多选
  3. Cmd+C 仿写
  4. iframe 内元素识别
  5. Popup 模式切换
- **关键路径覆盖**：每次提交前必须验证上述 5 条核心路径。
- **组件先行**：content.js 修改需在隔离测试页面先验证。

## 3. 禁止行为 (MUST NOT)
- 禁止未通过手动验证直接提交。
- 禁止使用用户正在浏览的页面作为测试环境（需独立测试页）。
- 禁止忽略 Lint / 编译错误。

## 4. 允许例外 (EXCEPTION)
- 纯视觉样式微调可放宽测试流程，但需人工 Review 确认。
- 文档更新无需测试验证。

## 5. 违规处理 (ENFORCEMENT)
- 未通过测试的 AI 改动禁止合并。
- 每次重大改动后必须输出测试记录：
  - 时间：YYYY-MM-DD HH:MM
  - 触发操作：
  - 覆盖路径：
  - 结果：
  - 异常：无/简述
